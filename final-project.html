User Research

User Interviews

User interview is an important process to get direct feedback from users. By asking user question about their expectation for certain website and their desire function of website, we can find the goal of the website and requirements from users in a considerably short time.


Competitor Analysis

Competitor identifies competitors and evaluating the strategies designers use for their design so that I can determine their strengths and weaknesses relative to those of my own design and try to use them or avoid them while I am designing my own work.


Feature Value Matrix

A feature Value Matrix helps us to evaluate features through a numerical comparison so that I can decide how important the fearture is. Features I choose to compare are frequently used by web designers. These common features are necessary, however, sometimes does not need for certain type of website.


Feature Prioritization

The feature prioritization matrix allow me to capture and prioritize feature requests and build out an effective and well-thought out personal profile website.


User Testing

User Stories

User story consists by a form of different people's expectation from the website. Different users want to get information they are finding for their own purpose. It captures the "who", "what" and "why" of a requirement in a simple, concise way, so that I can figure out both my targets and necessary information for the website.


Cognitive Walkthrough

The cognitive walkthrough method is a usability inspection method used to identify usability issues in interactive systems, focusing on how easy it is for new users to accomplish tasks with the system. The method is good for its ability to generate results quickly with low cost, especially when compared to usability testing, as well as the ability to apply the method early in the design phases, before coding even begins.

Summary of Findings

I found out that user research is a necessary process. These researches help me to collect requirements and necessary functions of a website in a very short time. Moreover, it helps with construct and design a brief view of the website, saving many times in design level.

After I finish the basic construction of the website I conducts user testing to figure out how current website satisfied users. I found out that various users having various requirements and asking for different information from me. Sometimes they prefer media elements rather than long paragraphs introducing.

The website that I created is too plain and simple, while users want to see more information related to my life and my professional experience.


Incorporation of Findings

In order to putting more element into the website, I spent a great amount of time in researching how to integrating media elements. I also separated my personal interests from the work experience and professional skills to satisfy different viewers.

In the future I might try to add some icons which relink to social media websites rather than keep users contact me only through email. Since the interviews states that they also want to connect me through social media, I will try to make the functions work in the next designing stage.
